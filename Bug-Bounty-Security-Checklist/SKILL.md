# Bug-Bounty-Security-Checklist - 安全检查清单技能

**来源**: Bug Bounty 课程学习提取
**版本**: 1.0
**创建时间**: 2026-04-02

---

## 功能描述

提供各类安全检查清单，帮助用户系统性地进行安全自查或渗透测试。

**支持场景**:
- Web 应用上线前检查
- API 安全检查
- 代码审查检查
- 服务器配置检查
- Bug Bounty 测试清单

---

## 触发条件

用户需要安全检查清单时触发，例如:
- "网站上线前需要做哪些安全检查"
- "API 安全怎么测试"
- "给我一份渗透测试清单"
- "代码审查需要注意什么"

## 示例对话

**用户**: 网站上线前需要做哪些安全检查？

**助手**:

```
# Web 应用上线前安全检查清单

## 使用说明
适用于 Web 应用正式上线前的安全自查，建议开发/测试/安全团队共同完成。

## 检查项

### 🔐 身份认证
- [ ] 密码策略 (最小长度 8 位，包含大小写/数字/特殊字符)
- [ ] 多因素认证 (MFA) 已实现
- [ ] 登录失败限制 (5 次失败后锁定 15 分钟)
- [ ] 会话超时 (30 分钟无操作自动登出)
- [ ] 安全退出功能 (清除会话)
- [ ] 密码重置流程 (Token 一次性/过期)
- [ ] 用户名枚举防护 (统一错误信息)

### 🔑 会话管理
- [ ] 会话 Token 使用加密随机数生成
- [ ] Cookie 设置 HttpOnly 标志
- [ ] Cookie 设置 Secure 标志 (仅 HTTPS)
- [ ] Cookie 设置 SameSite=Strict
- [ ] 会话固定攻击防护 (登录后更换 Session ID)
- [ ] 并发会话限制

### 🛡️ 输入验证
- [ ] 所有用户输入进行 XSS 防护
- [ ] 数据库查询使用参数化
- [ ] 无命令注入风险
- [ ] 文件路径进行规范化验证
- [ ] 文件上传验证类型/大小
- [ ] 表单包含 CSRF Token
- [ ] 输入长度限制

### 🔒 访问控制
- [ ] 每个功能点验证权限
- [ ] 无 IDOR 漏洞
- [ ] 敏感目录访问限制
- [ ] .git/.env 等文件不可访问
- [ ] API 端点访问控制

### 📦 数据保护
- [ ] 敏感数据加密存储
- [ ] 强制 HTTPS (HSTS)
- [ ] TLS 1.2+ 配置
- [ ] 密码使用 bcrypt/argon2 哈希
- [ ] API 密钥安全管理

### 🚨 错误处理
- [ ] 错误信息不泄露敏感数据
- [ ] 统一错误页面
- [ ] 错误日志记录
- [ ] 栈跟踪不暴露给用户

### 📋 安全 Header
- [ ] Content-Security-Policy
- [ ] X-Content-Type-Options: nosniff
- [ ] X-Frame-Options: DENY
- [ ] X-XSS-Protection: 1; mode=block
- [ ] Strict-Transport-Security

### 📝 日志监控
- [ ] 登录/登出日志
- [ ] 敏感操作日志
- [ ] 异常请求记录
- [ ] 告警机制配置

## 风险等级说明
- 🔴 高风险：发现即阻断上线 (如 SQL 注入/XSS)
- 🟡 中风险：建议上线前修复 (如信息泄露)
- 🟢 低风险：可排期修复 (如 Header 缺失)

## 下一步
1. 逐项检查并记录结果
2. 对发现的问题进行风险评级
3. 🔴 高风险必须修复后才能上线
4. 修复后复测验证
5. 形成安全检查报告
```
